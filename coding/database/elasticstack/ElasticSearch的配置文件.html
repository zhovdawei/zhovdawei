# 配置

# elasticsearch.yml

cluster.name 指定一个集群名
node.name 指定当前应用实例的节点名
network.host 绑定节点的IP
http.port 绑定节点的端口(除非默认端口9200被占用，否则不建议改)
path.data 指定当前节点的数据储存路径
path.logs 指定当前节点的日志存储路径


# 注意：如果network.host绑定了具体的IP。则应用以production模式启动，启动中
检查不满足生产要求，会出现Error报错，就像上一篇安装时的一些报错。
如果是localhost/127.0.0.1,则为Development模式，启动检查不符合要求，是warning警告日志。

# jvm.options

因为ElasticSearch是用Java开发的应用，所以我们需要关注虚拟机的资源分配和垃圾回收。 该配置文件需要关注的点：

1.堆的大小
官方的实例建议我们把堆的初始值和最大值配置一样，这样堆就是个固定值，这样有一个好处： 当堆内对象持续增长时可以避免扩容带来的full GC。

-Xms1g
-Xmx1g
1
2

2.设置GC

# 默认的GC配置
## GC configuration
# 使用ParNew + CMS + Serial Old 的收集器组合进行内存回收。
# Serial Old收集器将作为CMS收集器出现Concurrent Mode Failure失败后的后备收集器使用
-XX:+UseConcMarkSweepGC 
# 设置当老年代空间占用达到75%开始执行垃圾回收
-XX:CMSInitiatingOccupancyFraction=75
# 设置CMS只根据老年代空间占用指标来决定是否触发回收
-XX:+UseCMSInitiatingOccupancyOnly


# G1垃圾收集器配置，当你的JDK版本 >= 10
## G1GC Configuration
# NOTE: G1GC is only supported on JDK version 10 or later.
# To use G1GC uncomment the lines below.
# 10-:-XX:-UseConcMarkSweepGC
# 10-:-XX:-UseCMSInitiatingOccupancyOnly
# 10-:-XX:+UseG1GC
# 10-:-XX:G1ReservePercent=25
# 10-:-XX:InitiatingHeapOccupancyPercent=30

# 上面的垃圾回收参数可以自定义，比如JDK8就提供了G1，需要手动解锁开启
# 你可以忽略JDK版本统一使用G1

3.开启堆转储快照

# 官方是默认开启了，作用就是发生堆溢出时，保存了当时的现场情况，便于事后分析
