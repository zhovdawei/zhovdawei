# Centos7环境安装ElasticSearch

# 下载最新安装包


官网最新7.5.0(测试版)，官方也不建议我们在生产环境中使用它。不过我们是自己学习就不比担心。
我这里下载的是7.5.0版本，后续其他组件也是7.5.0版本。
推荐下载地址 (opens new window)
国内推荐复制下载链接到迅雷下，速度还挺快的。

# 暴露给外网访问

我是在虚拟机内操作的，如果要使用elasticsearch，需要暴露出来

# 首先关闭防火墙 或者 增加端口白名单，elasticsearch默认访问的端口：9200
# 我这里关闭防火墙
$ systemctl stop firewalld
$ systemctl disable firewalld
$ service network restart

# 可能出现的问题一：
[1]: max file descriptors [4096] for elasticsearch process is too low, increase to at least [65535]
# 解决方案：
[root@host114 elasticsearch-7.5.0]# vi /etc/security/limits.conf
################################################################
# 增加下面两行配置
* soft nofile 65536
* hard nofile 65536
################################################################

# 可能出现的问题二：
[2]: max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]
# 解决方案：
[root@host114 elasticsearch-7.5.0]# vi /etc/sysctl.conf 
################################################################
# 增加下面一行配置
vm.max_map_count=655360
################################################################
$ sysctl -p   使生效

# 可能出现的问题三：
[3]: the default discovery settings are unsuitable for production use; at least one of [discovery.seed_hosts, discovery.seed_providers, cluster.initial_master_nodes] must be configured
# 解决方案：
[root@host114 elasticsearch-7.5.0]# vi config/elasticsearch.yml 
################################################################
# 集群命名
cluster.name: zdw-cluster
# 节点命名
node.name: node-1
# ip绑定(你当前虚拟机的IP，如果你也是在虚拟机中操作)
network.host: 192.168.163.114
# 设置集群节点IP，目前就一台
discovery.seed_hosts: ["192.168.163.114"]
# 设置初始化master,目前就一台，就设置当前节点名
# 也可以不设置master,部署集群时，可以由集群间选举产生
cluster.initial_master_nodes: ["node-1"]

# 其他端口默认就是9200/9300.除非被其他占用，就无须改动
################################################################
