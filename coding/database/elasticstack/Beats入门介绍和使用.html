# Beats入门介绍和使用

# 几种Beats

官方介绍Beats为轻量级的数据采集器。
Filebeat 日志文件
Metricbeat 度量数据(主机的CPU,内存等使用情况，应用的监控指标)
Packetbeat 网络数据(抓包)
Winlogbeat windows系统上的数据
Heartbeat 健康检查

# 处理流程

1.输入 input
2.处理 filter
3.输出 output

Prospector(勘探者) : filebeat可以配置多个prospector,每个prospector对应一种文件匹配 规则。prospector能感知文件变化，进而触发harvester增量读取新的日志。
Harvester(收割者) : 每一个日志文件对应一个harvester,harvester将数据取出来发给output.

# 配置解析

1.input配置(yaml)
input_type: log(日志文件) / stdin(标准输入)

filebeat.prospectors:
  - input_type: log
    paths:
      /var/log/apache/httpd-*.log
  - input_type: log
    paths:
      - /var/log/messages
      - /var/log/*.log

# 高级用法

1.Elasticsearch Ingest Node
由于filebeat 缺乏数据转换能力，官方提供了ingest node。它是新的的node类型，可以在数据写入es前 对数据进行处理。它使用pipeline api。

2.Filebeat Module
filebeat把数据写入es要做一些配置文件的合理配置，还有ingest node中pipeline配置，还要对数据做 合理的kibana dashboard的配置。由于这些步骤繁琐复杂，官方推出了封装好的module,把常用的日志格式做统一处理。 比如：nginx / apache / mysql ...

# 介绍

Packetbeat用来实时抓取网络包数据，还可以将网络包按照网络协议展开解析。
支持的应用层协议：
ICMP(IPV4 / IPV6)
DNS
HTTP
MYSQL
REDIS

# 实战测试

1.创建测试配置文件
packetbeat.interfaces.device 监听的网卡(linux监听所有-any)
packetbeat.protocols.http 表示抓取http的包
ports 监听的端口
send_request true-开启记录http请求request的内容(body)
include_body_for 配置数据的筛选条件

[root@host114 packetbeat-7.5.0-linux-x86_64]# vi test.yml 
#######################################################################
packetbeat.interfaces.device: any
packetbeat.protocols.http:
  ports: [9200]
  send_request: true   
  include_body_for: ["application/json", "x-www-form-urlencoded"]

output.console:
    pretty: true
#######################################################################
