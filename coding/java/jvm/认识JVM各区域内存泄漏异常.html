# 虚拟机栈和本地方法栈溢出

“由于HotSpot虚拟机不区分虚拟机栈和本地方法栈，因此对于HotSpot来说，-Xoss参数 （设置本地方法栈大小）虽然存在，但实际上是没有任何效果的，栈容量只能由-Xss参数 来指定。”------来自《深入理解Java虚拟机》第三版。

# 异常之一

使用-Xss降低栈内存容量

/**
 * VM Args : -Xss128k
 * */
public class TestStackOverflow1 {

    private int stackLength = 1;

    public void stackLeak() {
        stackLength++;
        stackLeak();
    }

    public static void main(String[] args) throws Throwable{
        TestStackOverflow1 sof = new TestStackOverflow1();
        try {
            sof.stackLeak();
        }catch (Throwable e){
            System.out.println("stack length : "+sof.stackLength);
            throw e;
        }
    }
}

# 方法区和运行时常量池溢出

# JDK6和之前的常量池

JDK6和之前常量池分配在永久代中，可以通过 -XX:PermSize 和 -XX:MaxPermSize 限制永久代的大小。

/**
* VM Args : -XX:PermSize=6m -XX:MaxPermSize=6m 
* String::intern()是一个本地方法，如果string对象不在常量池，那么添加到常量池，返回引用
* 如果在常量池，直接返回引用
* */
public class RuntimeConstantPoolOOM{
    public static void main(String[] args){
        Set<String> set = new HashSet<String>();
        short i = 0;
        while (true){
            set.add(String.valueOf(i++).intern());
        }
    }
}

# JDK7方法区泄漏

使用CGLib增强，生成大量动态类。

/**
* VM Args : -XX:PermSize=10M -XX:MaxPermSize=10M
*/
public class JavaMethodAreaOutOfMemory{
    public static void main(String[] args){
        while (true){
            Enchancer enchancer = new Enchancer();
            enchancer.setSuperclass(OOMObj.class);
            enchancer.setUseCache(false);
            enchancer.setCallback(new MethodInterceptor(){
                public Object intercept(Object obj,Method method,Object[] args,
                MethodProxy proxy) throws Throwable{
                    return proxy.invokeSuper(obj,args);
                }
            });
            enchancer.create();
        }
    }
    static class OOMObj{
        
    }
}

# JDK8到目前(JDK13)

永久代移除，元空间登场，默认受限于本地内存大小。

-XX:MaxMetaspaceSize 设置元空间最大值,默认值-1(-1表示只受限制于本机内存大小),

-XX:MetaspaceSize 指定元空间的初始值大小，达到该值触发垃圾收集卸载类型

-XX:MinMetaspaceFreeRatio 垃圾收集后，控制最小空间剩余容量百分比

-XX:MaxMetaspaceFreeRatio 垃圾收集后，控制最大空间剩余容量百分比

# 本机内存直接溢出

/**
 * VM Args : -Xmx20M -XX:MaxDirectMemorySize=10m
 */
public class DirectMemoryOOM {

    private static final int _1mb = 1024 * 1024;

    public static void main(String[] args) throws Exception{
        Field unsafeFiled = Unsafe.class.getDeclaredFields()[0];
        unsafeFiled.setAccessible(true);
        Unsafe unsafe = (Unsafe)unsafeFiled.get(null);
        while (true){
            unsafe.allocateMemory(_1mb);
        }
    }
}
