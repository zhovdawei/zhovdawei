# Master-Slave

# 为什么引入Slave

jenkins可以帮助我们拉取代码，代码检查，打包...等等工作。当我们在一台 jenkins服务器上午去执行时，会排队等待。而且很多都是文件IO操作，耗费性能 大。所以我们需要将任务分配在slave上完成，以减轻master的压力。让master 做为任务的调度中心，让slave成为计算节点。

# 配置Slave


并发构建数：2 同一时间只允许两个job在构建。
远程工作目录：/opt/lib/jenkins 会在这个目录里面安装代理所需的jar,其他活动也在这个目录
用法：上面其实漏操作了，我们先用“只允许运行绑定到这台机器的JOB”测试，后面再改。
配置代理的主机名和账户密码：如果你的slave机的账户和密码都一样，配一个就行了。
高级栏目里面注意配置好该slave机上的JDK环境，比如我的：/usr/local/java/jdk1.8.0_231/bin/java
保存配置后，回到节点管理页面，可以点击节点名进去启动/断开。第一次连可能有点慢， 需要安装agent的环境，jar包等。

如上演示，我们指定slava-192.168.163.113这个名称的节点，jenkins就会去匹配。 如果状态正常就像上面动图中所示的那样。之后构建，我们可以看到日志打印中，有 构建的主机名，也有该主机上的工作目录输出。比如我这里：

# 指定slave标签构建


如上演示，给两台slave节点加上web标签。然后test任务指定节点为web标签。当对test 构建时，jenkins会从web标签中找到一组slave,并根绝jenkins自身的算法挑出一个，来 执行构建的具体任务。可以建多个job,同时去构建，可以看到job分在两个节点上执行。
