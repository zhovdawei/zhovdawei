# Jenkins+Docker自动化CI&CD流水线实战

# 目标流程


一共计划4个步骤：
拉代码->编译打包->制作镜像&推送->部署容器

需要注意的是，我这里项目结构是父子项目，子项目引用了父项目的配置。 而我的jenkins进入当前任务的目录为：
/var/lib/jenkins/workspace/web-1/
web-1是jenkins的任务名称，也是这个项目的父目录。 而我的子项目是java-web-1。所以在打包的命令上加了一句 cd java-web-1/ 以 便针对子项目打包。如果你不是这种结构，就不用理会。

# 构建镜像并推送到仓库

stage('镜像') {
    sh '''
        cd java-web-1/
        REPOSITORY=depo.zhoudawei.com/mytest/java-web-1:${branch}
        cat > Dockerfile << EOF
        FROM depo.zhoudawei.com/library/tomcat:v1 
        MAINTAINER zdw0607@gmail.com
        RUN rm -rf /usr/local/apache-tomcat-8.0.46/webapps/*
        ADD target/*.war /usr/local/apache-tomcat-8.0.46/webapps/ROOT.war
EOF
        docker build -t $REPOSITORY .
        docker login -udavid -pHarbor2019 depo.zhoudawei.com
        docker push $REPOSITORY
    ''' 
}

这里主要分布两步：
第一步：制作镜像docker build -t $REPOSITORY .
之前的章节我们介绍过镜像的三层结构，这里我们创建的是项目镜像，项目镜像就是包含了 运行环境和代码的镜像，直接跑起来就能访问到我们开发的程序。这里引用的tomcat镜像， 是运行环境镜像。Docker系列里面有完成的tomcat镜像制作和推送Harbor。
第二步：推送镜像docker push $REPOSITORY
首先你需要登录到Harbor服务器，然后就可以往上面推送了。
注意：这里需要注意一点EOF需要单独占一行，而且不能有空格。格式有些怪异， 主要是为了满足EOF格式需要。另外注意一下你tomcat镜像中，tomcat的安装目录。如果你安装完是默认目录， 可能就像我这样带有版本号的。那么你在编写dockerfile的时候，就要留意这个细节。

# 部署到测试服务器

stage('部署') {
    sh '''
        REPOSITORY=depo.zhoudawei.com/mytest/java-web-1:${branch}
        ssh root@192.168.163.106 "/usr/bin/docker rm -f java-web-1 |true"
        ssh root@192.168.163.106 "/usr/bin/docker pull $REPOSITORY"
        ssh root@192.168.163.106 "/usr/bin/docker container run -d --name java-web-1 -p 8090:8080 $REPOSITORY"
    '''
}

# 测试接口


106是docker容器运行主机，我们在这里部署项目。可以看到我们访问的是8090，这是在部署哪一步的脚本就确定了。 容器内tomcat默认运行的是8080端口，我们通过docker监听容器内端口8080，然后通过8090把项目暴露出去。 这样我们就以通过8090访问到我们的应用了。上面演示测试数据读写，也是正常。这说明在106上访问111mysql主机也是 正常的。
